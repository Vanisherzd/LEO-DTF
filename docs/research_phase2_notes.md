# LEO-DTF Research Phase 2 Notes

## Model-Driven Estimator Implementation

### Estimator Equations

The LEO-DTF estimator performs a grid search over 2D position (easting, northing) and time offset Δt, with closed-form least-squares estimation of nuisance parameters b₀ (CFO) and b₁ (drift).

#### Observation Model
The contaminated observations are modeled as:
- Frequency: z_f[t] = f_D[t] + b₀ + b₁·t + ε_f[t]
- Timestamp: z_τ[t] = t_nominal[t] + Δt + ρ[t]/c + ε_τ[t]

where:
- f_D[t] = -f_c/c · ρ̇[t] is the geometric Doppler shift
- ρ[t] is the geometric range
- ρ̇[t] is the geometric range rate
- ε_f[t] ~ N(0, σ_f²) and ε_τ[t] ~ N(0, σ_τ²) are measurement noises

#### Grid Search Formulation
The estimator computes a score for each candidate position p = [e, n] and time offset Δt:
S(p, Δt) = -log p(z_f, z_τ | p, Δt) - log p(b₀) - log p(b₁) - log p(Δt)

Under Gaussian assumptions, this becomes a weighted least-squares problem.

#### Closed-Form Nuisance Estimation
For fixed p and Δt, the nuisance parameters b₀ and b₁ can be estimated analytically by solving:
min_{b₀,b₁} Σ_w[f] (z_f[t] - f_D[t] - b₀ - b₁·t)² + Σ_w[prior] (b₀ - b₀,prior)²/σ_b₀² + Σ_w[prior] (b₁ - b₁,prior)²/σ_b₁²

This is solved via weighted least squares with virtual observations for priors.

#### Bayesian Posterior Scoring
After marginalizing over Δt using log-sum-exp for numerical stability:
p(p | z) ∝ exp(-S_MAP(p)) where S_MAP(p) = min_Δt S(p, Δt)

The posterior over the position grid is normalized and used to compute:
- MAP estimate: p_MAP = argmax_p p(p | z)
- HPD region: smallest region containing posterior mass ≥ threshold
- Ambiguity score: spread of significant modes in the posterior

**Important notes on interpreting results:**

1. **MAP single-run error is NOT a performance result** — The ~100 m error shown in smoke tests is a single Monte Carlo realization with arbitrary noise. It demonstrates code correctness, not achievable accuracy.

2. **CRLB is a theoretical lower bound** — The Cramér-Rao Lower Bound represents the best-case RMSE for any unbiased estimator under the assumed noise model. It is not a measured error.

3. **Monte Carlo required before paper claims** — To claim "X meter accuracy," the following are required:
   - 100+ independent trials with randomized noise
   - Report of mean RMSE with confidence intervals
   - Comparison of achieved RMSE to CRLB
   - Sensitivity analysis across geometry scenarios (elevation angles, pass duration)

4. **Grid resolution affects MAP estimate** — Current 20 m grid step quantizes position estimates. Finer grids reduce quantization error but increase computation.

### Current limitations

1. **Computational Complexity**: Grid search scales as O(G·D·N) where G is position grid points, D is time offset points, and N is number of observations. For fine grids, this can become expensive.

2. **Local Search Assumption**: The estimator assumes the true position lies within the predefined grid bounds. No hierarchical or adaptive refinement is implemented.

3. **Simplified Ambiguity Detection**: The ambiguity score uses a simple peak detection method that may not robustly identify multiple modes in complex posteriors.

4. **Flat Earth Approximation for Basis**: In the smoke test demo, an approximate ENU basis is used. For large baselines, the full basis computation from the reference geodetic point should be used.

5. **No Dynamic Updating**: The estimator processes a batch of observations. Sequential updating (e.g., particle filter) is not implemented.

6. **Prior Sensitivity**: Results can be sensitive to prior choices, particularly when observation geometry is poor.

### Smoke-Test Only Results

All results generated by the `run_smoke_test.py` script are synthetic and intended solely for:
- Verifying code correctness and numerical stability
- Testing that the posterior sums to 1
- Confirming the MAP estimate approaches the true position in high-SNR scenarios
- Validating that the estimator remains finite with nonzero CFO
- Ensuring utility functions (HPD region, ambiguity score) produce finite values

**No claims are made regarding:**
- Real satellite signal processing
- Actual hardware performance
- Paper-quality experimental results
- Comparison with existing methods
- Meter-level localization accuracy

The Phase 2 implementation provides the core algorithmic framework for future work with real data, but the synthetic demonstrations are strictly for development and validation purposes.

### CRLB timestamp sensitivity diagnostic

The CRLB sensitivity diagnostic (`scripts/diagnose_crlb_sensitivity.py`) quantifies how timestamp noise (σ_τ) propagates to position error bounds under fixed frequency noise (σ_f = 1.0 Hz). The diagnostic uses a 400 km circular equatorial orbit with 20 packets over 600 s and a ground station at 40.0°N, 105.0°W.

**Diagnostic results:**

| σ_τ (s) | range noise (km) | CRLB RMSE (m) | finite |
|---------|-----------------|---------------|--------|
| 1×10⁻³  | 299.8           | 413 565       | Yes    |
| 1×10⁻⁴  | 30.0            | 85 930        | Yes    |
| 1×10⁻⁵  | 3.0             | 13 540        | Yes    |
| 1×10⁻⁶  | 0.3             | 6 140         | Yes    |

Observations:

1. **Millisecond timestamping is too weak.** At σ_τ = 1 ms the implied range noise is ~300 km, which overwhelms the weak position sensitivity of propagation delay. The CRLB RMSE exceeds 400 km, confirming that millisecond-precision timestamps carry negligible geometric information for this estimator.

2. **Microsecond-level timestamping remains km-scale.** Even at σ_τ = 1 µs (range noise ~0.3 km), the CRLB RMSE is ~6 km. The ground-track geometry of a LEO pass at 400 km altitude limits the horizontal position information available from propagation delay alone; the horizontal dilution is substantial until the satellite rises to high elevation.

3. **MAP error is not comparable to classical CRLB.** The smoke-test MAP single-run error around 100 m reflects the Bayesian MAP estimator operating with priors on b₀, b₁, and Δt, plus grid discretization that regularizes the solution. The classical CRLB assumes an unbiased estimator with no priors. These two quantities are not directly comparable — the MAP estimator benefits from regularization that the CRLB does not account for.

4. **No paper performance claim is made from this diagnostic.** The diagnostic is a sensitivity sweep of a theoretical bound under simplified geometry. It establishes that timestamp precision matters and identifies the order of magnitude, but does not support an accuracy claim.

5. **Future validation should compare four quantities.** Before any paper claim:
   - Classical nuisance-marginalized CRLB (unbiased, no priors)
   - Prior-regularized / Bayesian FIM approximation (accounting for b₀, b₁, Δt priors)
   - Grid posterior covariance (numerical second moments of the discrete posterior)
   - Monte Carlo MAP error (100+ trials with randomized noise realizations)

These four should agree within Monte Carlo variance for the estimator to be well-characterized. Divergences indicate either prior misspecification or incorrect CRLB formulation.
